# Банкетное меню

Веб-приложение для автоматизации согласования банкетного меню между банкетным
залом и клиентом. Клиент по персональной ссылке самостоятельно выбирает блюда
и сразу видит итоговую стоимость мероприятия.

## Технологии

- **Next.js 15** (App Router) + **React 19** + **TypeScript**
- **Tailwind CSS** — светлая, минималистичная, адаптивная вёрстка
- **Next.js API Routes** — бэкенд
- **PostgreSQL** + **Prisma** — база данных и ORM
- **NextAuth** (Credentials) — авторизация администратора

## Возможности

### Администратор (банкетный зал)
- Вход в систему по email/паролю
- CRUD блюд: создание, редактирование, удаление, скрытие/активация
- Создание мероприятий с генерацией уникального токена и ссылки `/event/{token}`
- Просмотр всех мероприятий (название, клиент, дата, гости, стоимость, статус)
- Просмотр выбранного клиентом меню по каждому мероприятию

### Клиент (без регистрации)
- Открывает персональную ссылку `/event/{token}`
- Видит название, дату и число гостей
- Выбирает блюда по категориям (Закуски, Салаты, Горячие, Напитки, Десерты)
- **Мгновенный пересчёт** стоимости на гостя и общей суммы без перезагрузки
- Меняет количество гостей — итог пересчитывается автоматически
- Подтверждает меню → статус `Confirmed`, дальнейшее редактирование запрещено

Формула: **Стоимость банкета = Σ(цены выбранных блюд) × количество гостей**

## Статусы мероприятия

- **Draft** — клиент редактирует меню (выбор сохраняется автоматически).
- **Confirmed** — меню подтверждено, изменения заблокированы (на UI и на API).

## Быстрый старт

### 1. Зависимости
```bash
npm install
```

### 2. Переменные окружения
```bash
cp .env.example .env
```
Отредактируйте `.env` при необходимости. Обязательно задайте надёжный
`NEXTAUTH_SECRET` (например `openssl rand -base64 32`).

### 3. База данных PostgreSQL
Поднимите Postgres локально через Docker:
```bash
docker compose up -d
```
(или используйте свой сервер Postgres и пропишите `DATABASE_URL`).

### 4. Схема и сид-данные
```bash
npm run prisma:push   # создать таблицы по schema.prisma
npm run seed          # админ + каталог блюд + демо-мероприятие
```
Сид выводит в консоль учётку администратора и ссылку на демо-мероприятие.

### 5. Запуск
```bash
npm run dev
```
Откройте http://localhost:3000

## Структура проекта

```
prisma/
  schema.prisma            # Admin, Dish, Event, EventDish; enum Category, EventStatus
  seed.ts                  # админ + блюда + демо-мероприятие
src/
  app/
    page.tsx               # лендинг
    (auth)/login/          # вход администратора
    admin/                 # защищённая панель (layout с проверкой сессии)
      page.tsx             # таблица мероприятий
      events/[id]/         # детальная страница мероприятия (меню клиента)
      dishes/              # CRUD блюд
    event/[token]/         # публичная страница клиента (выбор меню)
    api/
      auth/[...nextauth]/  # NextAuth
      dishes/              # CRUD блюд (защищено)
      events/              # список/создание мероприятий (защищено)
      public/
        dishes/            # активные блюда для клиента
        events/[token]/    # GET/PATCH мероприятия по токену + confirm
  lib/                     # prisma, auth, validation, format, constants, event
  components/ui.tsx        # StatusBadge, Spinner
```

## Замечания по реализации

- **Деньги** хранятся в копейках (`Int`), чтобы избежать ошибок плавающей точки;
  форматирование — `Intl.NumberFormat('ru-RU')`.
- **Стоимость** на странице клиента считается мгновенно на клиенте, а изменения
  выбора/гостей сохраняются на сервер с debounce (500 мс). Источник истины при
  отображении в админке — серверный пересчёт из БД.
- **Блокировка после подтверждения** проверяется и на клиенте, и на сервере
  (PATCH/confirm возвращают `409`, если статус уже `CONFIRMED`).
- При сохранении выбора принимаются только существующие **активные** блюда.

## Ограничения MVP

Намеренно не реализованы: онлайн-оплата, PDF, складской учёт, закупки,
техкарты, роли пользователей, мультиресторанность. Один зал, один каталог.
